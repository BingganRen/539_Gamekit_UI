Usage Guide

Everything you need to drop GameKit UI into your own project — setup, theming, and accessibility notes.

Install Basic Usage Theming Accessibility About This Project

1. Install

GameKit UI has zero dependencies. There is nothing to install. Download the three core files from the component library and drop them into your project. 

your-project/
├── index.html
├── css/
│   └── style.css
└── js/
    └── main.js

Then link them from your HTML: 

<link rel="stylesheet" href="css/style.css">
<script src="js/main.js" defer></script>

2. Basic Usage

Every component is a plain HTML block. Copy the HTML from the component library, paste it into your page, and the CSS + JS in style.css and main.js will pick it up automatically.

Components are wired up via data- attributes. For example, a health bar starts at full HP and reads its max from the element: 

<div class="hp-bar" data-hp="100" data-hp-max="100"
     role="progressbar" aria-valuemin="0" aria-valuemax="100"
     aria-valuenow="100" aria-label="Player health">
  ...
</div>

To drive the bar from your game code, set the data attribute and re-render, or hook into the component by ID.

3. Theming

Every color, font, and spacing value is a CSS custom property. Override them in your own stylesheet after style.css loads: 

:root {
  --accent: #7c3aed;          /* purple instead of cyan */
  --accent-2: #f97316;        /* orange instead of pink */
  --bg: #0d0d0d;              /* pure dark */
  --font-display: "Orbitron", sans-serif;
}

All six components will pick up the new palette. There is no theme switcher built in — if you want a light theme, override the token values and the components will follow.

4. Accessibility

Every component was built with these commitments in mind:

  • All interactive controls are reachable by Tab and activate with Enter or Space.
  • Progress elements (HP, XP, cooldown) expose role="progressbar" and live aria-valuenow updates.
  • Color is never the sole carrier of meaning — HP state also pulses and changes labels.
  • Focus rings use :focus-visible and stay clearly visible on the dark background.
  • All animations respect prefers-reduced-motion.
  • Toast messages use aria-live="polite" so screen readers announce them without interrupting.
  • Text and background contrast has been validated against WCAG 2.1 AA.

5. About This Project

GameKit UI was built as the final project for SI 539: Design of Complex Websites at the University of Michigan School of Information, Winter 2026. It represents roughly 20 hours of solo work across research, design, implementation, and accessibility testing.

The goal was to practice three things at once: CSS animations (conic-gradient cooldowns, linear-gradient bars, shimmer effects), vanilla JS patterns (event delegation, data attributes, requestAnimationFrame), and accessibility in contexts where it is often ignored — game interfaces.

Source and build notes are on GitHub.
